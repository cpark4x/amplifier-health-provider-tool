## 🤝 Contributing to Documentation

### When to Update Documentation

- **Vision**: Only when strategic direction changes (rare)
- **Epics**: When planning major new capabilities
- **Templates**: Reference when creating new documents

### Creating New Documents

Use the templates in `docs/templates/`:
- Starting a new initiative? Use `VISION_TEMPLATE.md`
- Defining a feature? Use `EPIC_TEMPLATE.md`
- Planning implementation? Use `USER_STORY_TEMPLATE.md`

## 💡 Documentation Philosophy

This is a **lean** documentation structure:
- **Vision-driven**: Clear problem → solution → roadmap
- **Epic-focused**: High-level capabilities, not granular tasks
- **Template-guided**: Consistency without ceremony
- **Living documents**: Update as you learn, don't let them drift
